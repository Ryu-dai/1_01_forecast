# D2C 売上シミュレーター

プロダクト製造販売型 D2C スタートアップ向けの、売上・利益・必要販売数をインタラクティブにシミュレーションできる Web アプリです。

## セットアップ

### 必要な環境

| ツール | バージョン | 確認方法 |
|---|---|---|
| Node.js | **v20 推奨** | `node -v` |
| npm | v10+ | `npm -v` |

> **⚠️ Node v22 の注意**: `npx tsc` や `./node_modules/.bin/tsc` がシンボリックリンクの解決バグで動作しません。このプロジェクトでは型チェックに `node -e "require('./node_modules/typescript/lib/tsc.js')"` を使用してください。

### インストールと起動

```bash
git clone https://github.com/Ryu-dai/1_01_forecast
cd 1_01_forecast
npm install   # package-lock.json により全員が同一バージョンを取得
npm run dev
```

ブラウザで [http://localhost:3000](http://localhost:3000) を開いてください。

> **再現性を重視する場合**（CI環境など）は `npm ci` を使うと lock ファイルを厳密に適用できます。

### ビルド

```bash
npm run build
npm start
```

### 型チェック（ビルドなし）

```bash
node -e "require('./node_modules/typescript/lib/tsc.js')"
```

## 機能一覧

### 1. パラメータパネル（左サイドバー）

画面左側の固定サイドバーで全入力パラメータを設定します。スライダーをドラッグすると 150ms のデバウンスでリアルタイムにグラフ・カード・テーブルが更新されます。数値入力欄に直接タイプして Enter または フォーカスアウトすると即時反映されます。

#### プロダクト設定

| パラメータ | 範囲 | 初期値 | 説明 |
|---|---|---|---|
| 販売価格 | ¥500 〜 ¥50,000 | ¥5,000 | 1個あたりの販売単価 |
| 原価 (COGS) | ¥100 〜 ¥30,000 | ¥1,500 | 1個あたりの製造原価 |

パネル内に **粗利額** と **粗利率** がリアルタイム表示されます。

#### シミュレーション期間

**3ヶ月 / 6ヶ月 / 12ヶ月** をボタンで切り替えます。
期間を変えると認知度カーブの分布も再計算されます。

#### 目標利益

| パラメータ | 範囲 | 初期値 | 説明 |
|---|---|---|---|
| 目標利益 | ¥10万 〜 ¥5,000万 | ¥300万 | 期間内に達成したい総利益 |

目標利益と粗利から **必要総販売数** が逆算されます。

#### チーム / 製造キャパ

| パラメータ | 範囲 | 初期値 | 説明 |
|---|---|---|---|
| チーム人数 | 1〜20人 | 3人 | 製造に関わる人数 |
| 1人あたり製造数/月 | 10〜500個 | 100個 | 1人が月間に製造できる個数 |
| 週次製造キャパ上限 | 10〜2,000個/週 | 75個/週 | 週あたりの最大製造数（設備・外注制約など） |

チーム人数 × 月間製造数 と 週次上限の **小さい方** が実効キャパとして使われます。

#### 認知度カーブ

ブランド認知の広がりを4パターンから選択します。各オプションに **SVGスパークライン（折れ線プレビュー）** が表示されます。

| カーブ名 | 計算式 | 特徴 |
|---|---|---|
| **S字カーブ** | シグモイド関数 (k=1.5) | 中盤に一気に加速する典型的なグロースモデル |
| **線形** | 月番号に比例 | 毎月一定割合ずつ認知が広がるモデル |
| **指数** | e^(i×0.5) に比例 | 後半に向けて急増するバイラル型モデル |
| **イベントブースト** | S字 + 最終月×1.5 | 最終月にイベント等で認知が急伸するモデル |

いずれのカーブも **月別ウェイトの合計 = 1.0** に正規化されます。

#### チャネル配分

4つの販売チャネルの構成比を設定します（合計は常に 100% を維持）。

| チャネル | 色 | 初期値 | チャネル別マージン係数 |
|---|---|---|---|
| 直販 | 紫 | 40% | 1.0（基準） |
| 小売 | 緑 | 20% | 0.8 |
| EC | 黄 | 30% | 0.9 |
| 卸売 | ピンク | 10% | 0.6 |

スライダーを動かすと、変更したチャネル以外の3つが **現在の比率を維持したまま比例再配分** されます。各チャネルのマージン係数が粗利計算に反映されるため、卸売比率が高いほど有効粗利は低下します。

### 2. ダッシュボード（右メインエリア）

#### サマリーカード

6枚のカードが常時表示されます。

| カード | 内容 | 警告条件 |
|---|---|---|
| **必要総販売数** | 目標利益達成に必要な合計販売個数 | — |
| **必要週次ペース** | 期間全体を通じた平均週次販売個数 | 製造キャパ超過時に赤 |
| **必要日次ペース** | 期間全体を通じた平均日次販売個数 | — |
| **製造キャパ上限** | 設定した週次キャパ上限値 | 需要がキャパを超える月がある場合に赤 |
| **予測売上・粗利** | 総売上と総粗利の予測値 | — |
| **達成確度** | 製造キャパで達成できる利益 ÷ 目標利益（%) | 80%未満で赤 |

#### 月別グラフ（ComposedChart）

- **棒グラフ（黄）**: 月別販売数（個）
- **折れ線（紫）**: 月別売上（円）
- **折れ線（緑）**: 月別利益（円）
- **破線（薄紫）**: 累計利益（円）

左Y軸が販売数、右Y軸が金額です。

#### 週次ペースグラフ

- **折れ線**: 月別の週次平均販売ペース
- **赤破線**: 製造キャパ上限ライン
- キャパを超えた月のドットは **赤色** で表示されます

#### チャネル別円グラフ（ドーナツ）

期間合計の販売数をチャネル別に表示します。

#### 月別詳細テーブル

| 列 | 内容 |
|---|---|
| 月 | M1〜Mn |
| 認知度 | その月の認知度ウェイト（%） |
| 販売数 | その月の販売個数 |
| 週ペース | その月の週平均ペース（個/週） |
| 売上 | その月の売上額 |
| 利益 | その月の利益額 |
| 累計利益 | その月までの累計利益 |

製造キャパを超過している月の行は `bg-red-900/30`（暗い赤背景）でハイライトされ、「⚠ キャパ超過」バッジが表示されます。

### 3. シナリオ管理

最大 **3つ** のシナリオを名前を付けて保存し、比較できます。

#### 保存手順
1. パラメータを調整してシミュレーション結果を確認
2. 「+ 保存」ボタンをクリック
3. シナリオ名を入力して「保存」（名前省略時は「シナリオ N」が自動設定）

#### シナリオ比較ビュー
ヘッダーの「比較 (N)」ボタンをクリックすると、保存済みシナリオの並列比較テーブルが表示されます。

比較できる指標：
- 期間・販売価格・目標利益・認知カーブ
- 必要販売数・週次ペース
- 予測売上・予測利益
- 達成確度・キャパ超過の有無
- 予測利益の **横棒グラフ比較**（最大値を基準に相対表示）

---

### 4. ダーク / ライトモード

ヘッダー右上のアイコンボタンで切り替えます。設定は `localStorage` に保存され、次回アクセス時も引き継がれます。初回アクセス時はOS設定に従います。FOUC（スタイル適用前の白画面点滅）を防ぐインラインスクリプトを `<head>` に挿入しています。

## 計算ロジック

### 必要総販売数

```
有効粗利/個 = (販売価格 - 原価) × チャネル加重平均マージン係数
必要総販売数 = ⌈目標利益 ÷ 有効粗利/個⌉
```

### チャネル加重平均マージン係数

```
= (直販比率×1.0 + 小売比率×0.8 + EC比率×0.9 + 卸売比率×0.6) ÷ 100
```

### 月別販売数の配分

```
月別販売数[i] = round(必要総販売数 × 認知度ウェイト[i])
認知度ウェイト = 選択カーブで生成した数列を合計1.0に正規化
```

### 達成確度

```
実効週次キャパ = min(週次キャパ上限, チーム人数×月間製造数÷4.33)
最大可能利益 = 実効週次キャパ × 総週数 × 有効粗利/個
達成確度 = min(100, round(最大可能利益 ÷ 目標利益 × 100)) %
```
